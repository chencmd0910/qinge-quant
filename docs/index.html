💰 价格与充值

39AI 采用倍率计费模式，分组不同则倍率不同，倍率越高价格越高。所有模型输入输出价格同步各大模型官方计费逻辑，在官方基础上做折扣。

计费说明

网站充值比例：1 RMB = 1 USD

1 倍率 = 1 RMB = 1 USD（即冲 1 元可当官方的 1 美金使用）

扣费逻辑：每次请求费用 = 倍率 × 模型官方计费（Token 消耗量按官方标准）

例如：2 倍率分组下，1 美金的官方消耗 = 扣 2 元

余额不足时请求会被拒绝，请及时充值。

分组倍率说明

一个key全模型3×一个 Key 通调国内外各大模型，无需换 Key

gpt-image1×GPT Image 系列图片生成

充值说明

  1. 进入控制台 → 钱包管理，支持自助充值
  2. 点击「账单」可查看充值记录
  3. 点击「自定义」可输入任意金额充值，如遇白屏请关掉梯子或切换网络重试
  4. 如需开票，联系客服提供「用户名 + 开票信息 + 开票金额」，低于 1000 元作为无票收入入账，高于 1000 元可申请开票（金额可累计）
💳 支持支付宝、微信支付，进入控制台 → 充值即可。1 元起充，充值越多折扣力度越大！

平台会优先将请求打到主分组，失败后自动重试到备选分组，从而提升整体成功率。

🔌 API 参考

39AI 兼容 OpenAI API 格式，以下为常用接口说明。

Chat Completion

POST /v1/chat/completions 创建聊天补全 ▾

请求参数

参数类型必填说明
modelstring是模型名称，建议从模型广场复制
messagesarray是对话消息列表
streamboolean否是否流式输出，默认 false
temperaturenumber否采样温度，0-2，默认 1
max_tokensinteger否最大输出 Token 数

示例

Shell 
curl https://39ai.cc/v1/chat/completions \
  -H "Authorization: Bearer sk-xxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "deepseek-v4-flash",
    "messages": [
      {"role": "system", "content": "你是一个助手"},
      {"role": "user", "content": "介绍一下39AI"}
    ],
    "temperature": 0.7,
    "stream": true
  }'

🛠 客户端配置

39AI 兼容 OpenAI API，可在以下客户端中直接使用。

Cherry Studio

  1. 打开 Cherry Studio 设置 → 模型服务
  2. 选择 OpenAI API
  3. 填写 API 地址：https://39ai.cc
  4. 填写 API Key：你的 39AI 令牌
  5. 点击「添加模型」，输入你需要的模型名称
  6. 保存即可使用

Lobe Chat

  1. 打开 Lobe Chat 设置 → 语言模型
  2. 选择 OpenAI 提供商
  3. 配置自定义 API 地址：https://39ai.cc/v1
  4. 填写 API Key：你的 39AI 令牌
  5. 启用「自定义模型」并添加需要的模型
  6. 保存并开始对话

NextChat (ChatGPT Next Web)

  1. 打开 NextChat 设置
  2. 接口地址：https://39ai.cc
  3. API Key：你的 39AI 令牌
  4. 选择模型即可使用

OpenCat

  1. 打开 OpenCat 设置
  2. API URL：https://39ai.cc/v1
  3. API Key：你的 39AI 令牌
  4. 保存并选择模型

💻 开发接入

以下是使用 Python 和 Node.js 接入 39AI API 的示例代码。

Python 
from openai import OpenAI

client = OpenAI(
    api_key="sk-xxxxxxxxxxxxxxxxx",
    base_url="https://39ai.cc"
)

response = client.chat.completions.create(
    model="deepseek-v4-flash",
    messages=[
        {"role": "system", "content": "你是39AI助手"},
        {"role": "user", "content": "你好！"},
    ],
    temperature=0.7,
    stream=True,
)

for chunk in response:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="")
Node.js 
import OpenAI from 'openai';

const client = new OpenAI({
  apiKey: 'sk-xxxxxxxxxxxxxxxxx',
  baseURL: 'https://39ai.cc',
});

const response = await client.chat.completions.create({
  model: 'deepseek-v4-flash',
  messages: [
    { role: 'system', content: '你是39AI助手' },
    { role: 'user', content: '你好！' },
  ],
  temperature: 0.7,
  stream: true,
});

for await (const chunk of response) {
  process.stdout.write(chunk.choices[0]?.delta?.content || '');
}

❓ 常见问题

登录 39AI 官网控制台 → API 令牌 → 创建新令牌。创建后请妥善保管，令牌仅创建时可见。

1 倍率 = 1 元 = 1 美金。即充值 1 元可在官方计费中当作 1 美金使用。例如选择 2 倍率分组，官方消耗 1 美金时扣 2 元。分组倍率越高价格越高，但通常意味着更优质的渠道和更高的稳定性。

请检查：1) API Key 是否正确；2) API Key 是否在 Header 中以 Authorization: Bearer <key> 格式传递；3) API Key 是否过期或已删除。

支付宝/微信充值通常是即时到账。如遇白屏或延迟，请关掉代理或切换网络重试，仍不行则提供支付截图联系客服。

低倍率分组（如国产全模型 0.8×）性价比高，适合日常使用。高倍率分组（如官方 5×、企业推荐 8×）调用官方直连渠道，智力与稳定性最优。总结：99% 的场景低倍率就可以满足，只有对稳定性要求极高的场景才需要上高倍率。

在请求体中设置 "stream": true 即可启用 SSE 流式响应。各客户端通常支持自动识别流式模式。

支持。GPT-5.x、Claude Opus 4.6/4.7、Gemini 3.x 等模型均支持图片识别。在 messages 中使用 content 传入图片 URL 或 Base64 即可。

余额不足时请求会被拒绝。请及时充值以避免服务中断。

联系客服提供「用户名 + 开票信息 + 开票金额」即可。低于 1000 元作为无票收入入账，高于 1000 元可申请开票，金额可累计。

根据账户等级有所不同。基础用户每分钟 60 RPM，高级用户请查看控制台套餐详情。
